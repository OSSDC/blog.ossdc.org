A new OSSDC.org MVP to build a smart dash camera using Android phones

Posted on April 29, 2017April 29, 2017 by Marius Slavescu

The idea is to create an Android app that can be used to easily collect datasets for OSSDC.org development/testing, also test in realtime simple ADAS functions.

We will develop this app as part of our MVP 4 project:

https://github.com/OSSDC/OSSDC-DashCamera

On powerful phones like this we can run pretty advanced algorithms, you can get it for less then $200 USD, has a great camera (no OIS though), IMU sensors and flagship Snapdragon 820 CPU/GPU:

http://m.gsmarena.com/leeco_le_max_2-8051.php

I got exactly this model:

http://m.geekbuying.com/item/LeTV-LeEco-Le-Max-2-X829-4GB-64GB-Smartphone—Gray-376308.html

I forked several Android related projects, in https://github.com/OSSDC, for OBD2 integration, would be good to see how we pull them into a unified app, based on VideoSensors.

Currently I use Torque and OpenCamera (because I can control focus, I’ll switch back to VideoSensors when we have that feature there) to collect datasets, but we need a simple setup/app to do this.

Initially we should extend this project to add focus control, better exposure, and OBD2 Bluetooth collection integration: 

https://github.com/OSSDC/VideoSensors

Then integrate something like:

https://github.com/OSSDC/android-robocar

https://twitter.com/GTARobotics/status/855952341627006976

More integration ideas here, these could be used for (see OSSDC Slack) #toy_sdc #mid_size_sdc also:

https://platis.solutions/blog/2015/06/29/worlds-first-android-autonomous-vehicle/

Also people and car detection like in Tensorflow Android demo:

https://github.com/tensorflow/tensorflow/tree/master/tensorflow/examples/android

Join our efforts in Open Source Self Driving Car Initiative at:

http://OSSDC.org
